HR Platform Design System & Visual Redesign

2026-03-20 Draft CALATRAVA — Coastal Debt HR Platform

Redesign the HR platform's visual identity from a dark-first glassmorphism aesthetic to a soft, warm, light-only design inspired by the Coastal Debt brand book. Establish a token-level design system as the foundation for all UI.

Goals

  1. Visual cohesion — Replace inconsistent glassmorphism/gradient style with a unified warm, approachable look
  2. Brand alignment — Adapt Coastal Debt brand colors and typography for an internal HR tool
  3. Maintainable tokens — Single source of truth for colors, typography, spacing, shadows, and radii as CSS custom properties
  4. Smooth migration — Components already consume CSS variables via Tailwind, so token changes cascade naturally

Non-Goals

  • Building a documented component library (tokens only)
  • Adding new features or changing functionality
  • Changing component architecture (CVA + Tailwind stays)
  • Changing layout structure (sidebar + topbar + mobile nav stays)

Design Direction

Style

What Stays

  • Component architecture — CVA variants, Tailwind utilities, cn() helper
  • Layout structure — Sidebar + TopBar + MobileNav
  • Animation system — Framer Motion for transitions, stagger, counters
  • Animations — fadeUp, pulse-slow keyframes (remove glow-pulse only)
  • Icon library — Lucide React (already line-style per brand guidelines)
  • All functionality and routing — Zero functional changes

Migration Strategy

  1. Update tokens in globals.css — most changes cascade automatically
  2. Search and replace glass/gradient/glow classes (~30 files)
  3. Replace inline purple gradients — 11+ files, replace with solid bg-accent
  4. Remove theme infrastructure — ThemeProvider, toggle, dark selectors
  5. Add Aeonik font — font files + @font-face + global heading rule
  6. Audit and adjust — verify each page looks correct with new tokens
This is a visual-only change. No database, API, or business logic modifications.
